GitHub Setup &
Effective Workflow

How to set up GitHub properly, configure branches, follow conventional commits, and use GitHub Actions CI/CD.

🔧

First-Time GitHub Setup

Connect your local Git to your GitHub account securely once per machine.

1

Create a GitHub Account

Sign up at github.com with a professional username.

2

Configure Identity Locally

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
3

Set Up SSH Authentication

ssh-keygen -t ed25519 -C "you@example.com"
cat ~/.ssh/id_ed25519.pub | clip
🌿

Branching Strategy & Conventional Commits

Never commit directly to main. Use structured branch names and commit prefixes.

feat

New feature added

feat: add user authentication flow
fix

Bug fix referencing issue

fix: resolve nav overflow on mobile (#12)
style

CSS styling / formatting

style: update card hover transition timing
refactor

Code restructuring

refactor: extract Button component
AI-First Development

AI Context Files &
Knowledge Management

How to write AGENTS.md, maintain a docs/ folder, and give AI assistants maximum project context.

🧠

Without Context Files

AI has zero memory between sessions. You re-explain your stack, conventions, and architecture every single time.

⚡

With Context Files

AI reads AGENTS.md, instantly understands your architecture, and produces on-brand code from message one.

📄

AGENTS.md Template Structure

Keep AGENTS.md in your project root as the AI assistant briefing document.

# AGENTS.md — AI Assistant Briefing

## Project Overview
- Name: MyApp
- Tech Stack: HTML, CSS, JS, Node
- Folder Structure: assets/, css/, js/, pages/

## Coding Standards
- Use BEM naming in CSS
- TypeScript strict mode enabled
- Never hardcode secrets in source code
