Before You Code

Pre-Project Setup
Checklist

The 8 essential steps to complete in your workspace before writing a single line of code. Skipping these is how projects go sideways.

01
📁

Set Up Folder Structure

Organise your project before any files are created. A clear directory layout (assets/, css/, js/, pages/) keeps work scalable and prevents root-folder chaos.

my-project/
├── index.html
├── README.md
├── assets/
│   ├── images/
│   └── fonts/
├── css/
│   └── style.css
└── js/
    └── script.js
OrganisationScalability
02
📝

Create a README.md

Document your project from day one. A good README covers: purpose, features, local setup, tech stack used, and key resources.

  • Project name & description
  • Core features list
  • How to run locally
  • Technologies used
  • Screenshots
DocumentationOnboarding
03
🎯

Define the Project Goal

Before touching code, answer core questions in writing. Vague goals lead to endless rewrites and scope creep.

  • What problem does this solve?
  • Who is the target audience?
  • What are v1 core features?
  • What design style?
StrategyScope
04
🎨

Plan Your Design System

Define CSS custom properties (design tokens) at the top of your stylesheet before writing components.

:root {
  --color-primary: hsl(250,80%,60%);
  --color-bg: hsl(220,20%,8%);
  --font-main: 'Inter', sans-serif;
  --space-md: 1rem;
  --radius-md: 12px;
}
Design TokensCSS
05
🔤

Choose & Import Fonts

Pick a Google Font pair (e.g. Outfit + JetBrains Mono) and add preconnect links before writing CSS.

<link rel="preconnect" href="https://fonts.googleapis.com">
<link href="https://fonts.googleapis.com/css2?family=Outfit:wght@400;600;700&display=swap" rel="stylesheet">
TypographyBranding
06
🌐

Set Up index.html Properly

Start with a solid HTML5 boilerplate including viewport, meta description, and proper semantics.

  • lang attribute on <html>
  • Viewport meta tag set
  • Descriptive title tag
  • Meta description for SEO
SEOHTML5
07
🔀

Initialise Git

Run git init before writing code. Create a .gitignore immediately to keep secrets out of history.

git init
git add .
git commit -m "initial setup"

# .gitignore:
node_modules/
.env
GitVersion Control
08
📐

Wireframe the Layout

Sketch layout blocks before writing HTML. Wireframing reveals layout problems before spending hours coding.

  • Map out sections
  • Define navigation structure
  • Plan visual hierarchy
  • Identify reusable components
UXPlanning
Ship It Right

Pre-Launch
Checklist

The non-negotiable verification steps required before your project goes live in production.

🔍

SEO & Meta

  • Unique title tag (<60 chars)
  • Meta description (150–160 chars)
  • Open Graph tags: og:title, og:image
  • Canonical URL set
  • robots.txt & sitemap.xml generated
⚡

Performance

  • Lighthouse score ≥ 90 on all 4 metrics
  • LCP < 2.5s, FID < 100ms, CLS < 0.1
  • Images compressed to WebP
  • Fonts preloaded with rel="preconnect"
♿

Accessibility

  • Images have descriptive alt text
  • Colour contrast ≥ 4.5:1 (WCAG AA)
  • Full keyboard navigation (Tab through)
  • Visible focus indicators on interactive controls
🔒

Security & UX

  • HTTPS enforced & HSTS header set
  • No secrets in client-side code or Git
  • Mobile responsive verification (320px to 4K)
  • Custom 404 page configured
